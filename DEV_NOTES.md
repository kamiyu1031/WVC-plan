# 開發與修正紀錄 (Development & Correction Notes)

本文件紀錄了 WVC-plan 在 2026-03-06 的開發過程與錯誤修正紀錄。

## 1. 任務精靈 (Task Wizard) 重構
- **需求**：將原有的純文字任務輸入框，改為可以用選單「業主 -> 標案 -> 縣市區域 -> 地址」多步驟操作的任務精靈。
- **實作**：
  - 在 `index.html` 內新增了 Wizard 介面與狀態管理（`wizardStep`, `wizardData`）。
  - 新增上傳 Excel 任務清單的功能（`handleMissionExcelUpload`）。
  - 將台灣所有縣市與區域資料寫入常數 `TW_AREAS`（並修復「中山區」文字解析錯誤的編碼問題）。

## 2. 預載入預設清單
- **需求**：每次重整網頁時必須重新匯入 Excel 才能進行任務精靈和工班人員選擇，希望可以直接寫入腳本。
- **實作**：
  - 獨立使用 Node.js 的 `xlsx` 套件讀取 `260303-workers list.xlsx` 及 `mission-list-copy.xlsx`。
  - 將解析出來的 JSON 資料寫成固定的設定值，直接成為 `DEFAULT_WORKERS` 與 `missionConfig` 的預設狀態。
  - **修正過程**：在抓取 `260303-workers list.xlsx` 內容時，由於原程式碼抓取了錯誤的試算表及欄位，導致人員簡稱顯示為 "0" 或抓成不完整的字串。經由撰寫獨立腳本重新抓取「工班清單」工作表的 B欄（姓名）與 C欄（簡稱），成功建立完整的預設 JSON 陣列。

## 3. 上班時間彈性輸入與文字排班匯出
- **需求**：
  1. 日期標題側邊需要新增可以調整「早班時間」與「晚班時間」的欄位。
  2. 視窗下方需要一個「複製文字班表」的按鈕，按下後依據上面設定的時間、有排班的對象與其簡稱進行字串組合與剪貼簿複製。
- **實作與錯誤排解**：
  - 新增了 `shiftTimes` 狀態用來記憶每日的早晚班時間，並加入資料的 Save / Load 機制。
  - 實作了 `handleCopySchedule` 去組合文字。
  - **Bug發生**：一開始使用寫死的系統 key 值（`workerStatus === 'morning'`）進行文字輸出判斷，但因自訂狀態上傳 Excel 後，系統狀態轉成了 `status_1` 等字樣導致判斷失效。
  - **解決方案**：將判斷機制改為比對 `statusTypes[workerStatusKey].label` (標籤名稱)，若名稱中含有「早班」「連班」或「晚班」則直接配對輸出，徹底解決自訂標籤失效的問題。
